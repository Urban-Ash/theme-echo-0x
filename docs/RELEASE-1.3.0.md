# Echo 0x 1.3.0

Echo 0x `1.3.0` 把前几个迭代中的首页、文章阅读、主题配置与 Halo 插件适配整理为一次完整发布。首页回归博客索引，文章封面和目录获得更稳定的加载与阅读体验，友情链接也开始显示基于访客网络环境的可达性参考。

## 亮点

- 首页使用 Halo 站点名称作为主标题，三段自定义文案作为站点标语
- 首页索引数据、文章卡片封面策略、导航信息和页脚项目均可配置
- 无主图文章按标题生成稳定的公式封面；有主图时先显示 SVG 动画占位
- 文章页提供可关闭的左侧章节导航与当前章节跟踪
- Links 页面自动标记 `online / warn / idle`，不再使用模拟状态
- 支持跟随系统、明亮、黑暗三种模式，并允许修改辅助色
- 接入 Halo 搜索、评论、图库正文类和页脚扩展点
- 404 页面采用与主题一致的 `$ resolve "/path"` 终端诊断样式

## 配置变化

后台设置现分为六组：

- `home`：首页标语、简介、索引数据和文章卡片封面策略
- `style`：辅助色、明暗模式、页面切换和公式封面风格
- `nav`：时间、访客 IP、动态页面路径和 Halo 菜单
- `plugin`：搜索入口与评论组件
- `post`：章节导航和文章主图策略
- `footer`：cat 文案、暂停动画、RSS、GitHub 与 Email

页面切换动画在 `1.3.0` 中默认关闭；启用后使用轻量淡入效果。访客 IP 默认关闭，其余导航基础信息按设置决定是否显示。

## 友情链接状态

Links 插件模式会从当前访客浏览器发起两次轻量探测：

- `online`：两次成功且响应较快
- `warn`：响应慢、结果不稳定，或浏览器无法可靠验证
- `idle`：两次失败、超时或协议不受支持

此状态会受到跨域规则、网络代理和目标站点策略影响，仅作为当前访问环境的参考。

## 安装

1. 下载 `1.3.0.zip`
2. 进入 Halo 后台的“主题”页面
3. 上传压缩包并启用 `Echo 0x`
4. 进入主题设置，确认首页标语、导航菜单、插件开关和页脚链接

## 升级说明

- 本版本要求 Halo `2.25.0+`
- 从 `1.2.x` 升级后建议重新上传 ZIP，以刷新带版本号的 CSS 与 JavaScript
- 搜索、评论和图库能力需要安装对应 Halo 插件；未安装时主题仍可正常使用
- 视觉效果需要在实际 Halo 内容与插件环境中完成最终确认
